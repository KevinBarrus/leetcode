链接：[25. K 个一组翻转链表 - 力扣（LeetCode）](https://leetcode.cn/problems/reverse-nodes-in-k-group/description/?envType=study-plan-v2&envId=top-100-liked)

给你链表的头节点 `head` ，每 `k` 个节点一组进行翻转，请你返回修改后的链表。

`k` 是一个正整数，它的值小于或等于链表的长度。如果节点总数不是 `k` 的整数倍，那么请将最后剩余的节点保持原有顺序。

你不能只是单纯的改变节点内部的值，而是需要实际进行节点交换。

**示例 1：**

![](https://assets.leetcode.com/uploads/2020/10/03/reverse_ex1.jpg)

**输入：head = `[1,2,3,4,5]`, k = 2
输出：`[2,1,4,3,5]`

**示例 2：**

![](https://assets.leetcode.com/uploads/2020/10/03/reverse_ex2.jpg)

输入：head = `[1,2,3,4,5]`, k = 3
输出：`[3,2,1,4,5]`

**提示：**

- 链表中的节点数目为 `n`
- `1 <= k <= n <= 5000`
- `0 <= Node.val <= 1000`

进阶：你可以设计一个只用 `O(1)` 额外内存空间的算法解决此问题吗？

### 方法一：递归

此题其实还是翻转链表的问题，只不过翻转的个数变成了k个，只需要加一个用来计数的变量即可

翻转方法，以 A -> B -> C 为例

```
pre   cur
A  ->  B   ->  C
head
```

翻转分两步：

B - A - C

C - B - A

第一步中，我们把 cur 对应节点扔到表头

如果我们让 pre cur 向后滑动一个节点，那么第一步完成后，pre 指向 A，cur 指向 C

那么在第二步中，我们依然把 cur 对应节点扔到表头

第二步完成后，cur 为 null，翻转完毕

翻转代码：

Java
```java
        ListNode pre = head;
        ListNode cur = head.next;
        while(cur != null) {
            //A -> B -> C，以第一轮为例，pre 为 A，cur 为 B
            pre.next = cur.next; //A -> C
            cur.next = head; //B -> A, 把 cur 对应节点扔到链表头，完成这一步后 head 不是表头，要调整
            //走到这里，实际上顺序已经变成了 B -> A -> C，完成了第一步，接下来调整 pre cur 和 head
            head = cur ; //保证 head 始终是表头，这样后续的 cur 才能扔到 head 前面，即表头
            cur = pre.next;
        }
```

现在只需翻转 k 个，那么新增一个计数变量即可

```java
        ListNode pre = head;
        ListNode cur = head.next;
        int cnt = 0;
        while(++cnt < k && cur != null) {
	        //因为k个节点只需要把k-1个节点扔到前面，所以是++cnt而不是cnt++
            //A -> B -> C，以第一轮为例，pre 为 A，cur 为 B
            pre.next = cur.next; //A -> C
            cur.next = head; //B -> A, 把 cur 对应节点扔到链表头，完成这一步后 head 不是表头，要调整
            //走到这里，实际上顺序已经变成了 B -> A -> C，完成了第一步，接下来调整 pre cur 和 head
            head = cur; //保证 head 始终是表头，这样后续的 cur 才能扔到 head 前面，即表头
            cur = pre.next;
        }
```


有多组，所以不能只处理一轮。对于一组来说，翻转完成后 pre 为这一组最后一个节点，cur 为最后一个节点的下一个节点，因此对于下一组来说，cur 就是它们的 head，递归调用即可

但是需要注意多组之间的连接关系。下一组翻转完成后，上一组的最后一个节点应该指向下一组的头。每一组翻转完成后，尾节点都是 pre，头节点都是 head，所以最终返回值应该是这一组的头节点 head

如果最后一组的节点数量小于k怎么办？

首先，怎么判断节点数量？通过 cnt 可以知道，而 cnt 是一边翻转一边增加的，也就是说不管这一组的节点数量是否小于 k ，翻转都是不可避免的

所以，可以先翻转，然后判断 cnt 是否小于 k，如果是，说明节点数量小于 k，那么就再翻转一遍。head 的作用就凸显出来了，因为对于一组来说，head 永远是这一组节点的头

Java

```java
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        if(head == null) return null;
        ListNode pre = head;
        ListNode cur = head.next;
        int cnt = 0; //表示 k 个中已经扔到前面的节点
        while(++cnt < k && cur != null) {
            //因为k个节点只需要把k-1个节点扔到前面，所以是++cnt而不是cnt++
            //A -> B -> C，以第一轮为例，pre 为 A，cur 为 B
            pre.next = cur.next; //A -> C
            cur.next = head; //B -> A, 把 cur 对应节点扔到链表头，完成这一步后 head 不是表头，要调整
            //走到这里，实际上顺序已经变成了 B -> A -> C，完成了第一步，接下来调整 pre cur 和 head
            head = cur; //保证 head 始终是表头，这样后续的 cur 才能扔到 head 前面，即表头
            cur = pre.next;
        }
        if(cnt < k) {
            //再翻转一次
            return reverseKGroup(head, cnt); //head永远是这一组的头节点
        }
        pre.next = reverseKGroup(cur, k); //上一组的尾节点应当指向下一组翻转完成后的头节点 head，所以 reverseKGroup 应当返回 head
        return head;
    }
}
```

时间复杂度：n/k组，每组移动 k-1 次节点，时间复杂度为O(n)

空间复杂度：递归深度 n/k，每层递归 O(1) 空间，空间复杂度为 O(n/k)

### 方法二：迭代

方法一空间复杂度之所以不为O(1)，正是因为进行了递归。而之所以使用递归，是因为 pre cur 这对双指针只对一组生效。想办法让双指针全局生效

就迭代法翻转链表来看，实际上是非常简单的。就是 pre 始终为 cur 的前驱节点，然后调整 pre 和 cur 的指向关系，再一起往后移动。

至于k个一组，由于上一种方法是以组为单位递归的，因此可以通过判断一组节点个数是否小于k来判断最后一组节点数量是否小于k，但在迭代法中并不方便，因为还要考虑处理最后一组已经被翻转的问题。所以不妨先统计节点的数量，每次遍历一组，n自减k，直到n小于k时停止循环，这样就避免翻转了节点数量小于k的最后一组

迭代法翻转至少需要三个变量： 前驱节点 pre + 当前节点 cur + 后驱节点 nxt，先让 cur 指向 pre，这样就实现了 pre  cur 之间指向的翻转。再把 cur 的位置给 pre，nxt 的位置给 cur，相当于 pre 和 cur 后移，再次进行翻转，一直到表尾

重点在于，翻转一组之后，如何与前一组、后一组进行连接的问题，假设链表一开始为：
```
1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7 -> 8 -> 9
```

且 k = 3 

考虑一般性，假设现在第一组已经完成了翻转，处理第二组。即此时链表为：
```
3 -> 2 -> 1 -> 4 -> 5 -> 6 -> 7 -> 8 -> 9
```

对于迭代法翻转链表，起初 pre 应当为 null，cur 应当为头节点，所以此时 pre 为1，cur 为4

翻转完第二组后：

```
                         pre cur
3 -> 2 -> 1 -> 4 <- 5 <- 6 -> 7 -> 8 -> 9
```

注意，这里4的 next 为 null，因为只是组内翻转，不涉及到与前一组的连接问题

不难发现，为了与上一组建立正确的连接，上一组的尾节点也应当记录下来。即1应当指向6

同时，为了与下一组建立正确的连接，本组的头节点（未翻转前的）也应当记录下来，即4应当指向7

为了统一管理，我们事先创建一个虚拟头节点 dummy，这样就能保证各组的逻辑是一致的，不需要针对第一组而写单独的代码，即链表一开始为：
```
d -> 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7 -> 8 -> 9
```

对于第一组，组内翻转完但未与其它组建立正确连接前，dummy 充当的就是“上一组的尾节点”的角色，但 dummy 实际上是永远指向3的，而对于其它组来说，“上一组的尾节点”是不固定的，因此我们需要事先声明一个 p 指向 dummy，这样就分离了 dummy 和 “上一组的尾节点”这两个角色，统一用 p 表示“上一组的尾节点”

所以，翻转完第二组后：
```
               p             pre  cur
d -> 3 -> 2 -> 1 -> 4 <- 5 <- 6 -> 7 -> 8 -> 9
```

上一组的尾节点已经有了，本组的头节点（未翻转前的）也有了，就是 p.next

因为接下来要让 p.next 为 7，所以事先用一个变量 cgt(current group's tail)记录当前组的尾节点

```
               p   cgt       pre  cur
d -> 3 -> 2 -> 1 -> 4 <- 5 <- 6 -> 7 -> 8 -> 9
```

第一步：由于翻转前的头节点翻转后成为了本组的尾节点，因此需要让它指向下一组的头节点
让4指向7，即 cgt.next = cur

```
               p   cgt       pre  cur
d -> 3 -> 2 -> 1 -> 4 <- 5 <- 6 -> 7 -> 8 -> 9
                    |              ⬆
                    |              |
                    ----------------
```

第二步：由于翻转前的尾节点翻转后成为了本组的头节点，因此需要让上一组的尾节点指向它
让1指向6，即 p.next = pre

```
               p   pre       cgt  cur
d -> 3 -> 2 -> 1 -> 6 -> 5 -> 4 -> 7 -> 8 -> 9
```

现在就已经完全正确与前后组建立连接了！下一组就是 7 8 9 了

但 p 的作用是”上一组的尾节点“，对于 7 8 9 来说，”上一组“指的是 4 5 6，尾节点为4，所以还要让 p = cgt

接下来就是如法炮制翻转 7 8 9 了

不过，为了方便理解，在进行下一组的翻转之前，可以将 pre 置为 null，表示”为了避免本组翻转时第一个节点与第二个节点互相指向，初始的 pre 已充当第一、二个节点翻转后，第一个节点的后驱节点“，实际上这行代码可有可无，仅仅只是方便理解

```java
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        //先统计节点个数 n
        int n = 0;
        for(ListNode tmp = head; tmp != null; tmp = tmp.next) {
            n++;
        }

        //创建虚拟头节点，充当第一组的“上一组的尾节点”角色，以统一代码逻辑
        ListNode dummy = new ListNode(0, head);

        //创建p，统一表示所有组的”上一组的尾节点”
        ListNode p = dummy;

        //迭代的前驱节点 pre，对于每一组，开始翻转前 pre 统一为 null
        ListNode pre = null;

        //迭代的当前节点 cur
        ListNode cur = head;

        //k个一组，每次让n自减k，直到n不大于k时停止循环，因为此时必然进入最后一组且最后一组节点数量小于k
        for(; n >= k; n -= k) {
            //内层循环针对某一组，直接从第一个遍历到第k个
            for(int i = 0; i < k; i++) {
                //开始翻转，调整指向，双指针同步右移
                ListNode nxt = cur.next;
                cur.next = pre;
                pre = cur;
                cur = nxt;
            }

            //此时 pre 为本组尾节点（翻转前的），cur 为下一组的头节点

            //由于翻转前的头节点翻转后成为了本组的尾节点，因此需要让它指向下一组的头节点
            ListNode cgt = p.next; //本组翻转前头节点
            cgt.next = cur;

            //由于翻转前的尾节点翻转后成为了本组的头节点，因此需要让上一组的尾节点指向它
            p.next = pre;

            //即将进入下一组，则“上一组的尾节点”就是 cgt(current group's tail)，调整 p
            p = cgt;

            //下一组翻转前，保持 pre 置空
            pre = null;
        }

        return dummy.next;
    }
}
```
