给定两个字符串 `text1` 和 `text2`，返回这两个字符串的最长 **公共子序列** 的长度。如果不存在 **公共子序列** ，返回 `0` 。

一个字符串的 **子序列** 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。

- 例如，`"ace"` 是 `"abcde"` 的子序列，但 `"aec"` 不是 `"abcde"` 的子序列。

两个字符串的 **公共子序列** 是这两个字符串所共同拥有的子序列。

**示例 1：**

**输入：**text1 = "abcde", text2 = "ace" 
**输出：**3  
**解释：**最长公共子序列是 "ace" ，它的长度为 3 。

**示例 2：**

**输入：**text1 = "abc", text2 = "abc"
**输出：**3
**解释：**最长公共子序列是 "abc" ，它的长度为 3 。

**示例 3：**

**输入：**text1 = "abc", text2 = "def"
**输出：**0
**解释：**两个字符串没有公共子序列，返回 0 。

**提示：**

- `1 <= text1.length, text2.length <= 1000`
- `text1` 和 `text2` 仅由小写英文字符组成。


子序列问题中，我们一般都会设`dp[i]`为字符串0~i部分最长公共子序列的长度

这里有两个字符串，要求它们的公共最长子序列，可以令`dp[i][j]`表示`text1[0:i]`与`text[0:j]`的最长公共子序列。这里`text[0:k]`表示从下标0的元素到下标k的元素组成的子串

首先考虑递推公式，不难看出，关键在于`text1[i]`和`text2[j]`是否选

所谓是否选，就是看其存在是否会影响到公共子序列的长度。显然，如果`text1[i]`等于`text2[j]`，那么它们两个必然可以同时为“增长最长公共子序列”这件事情贡献力量，因为它们是公共子序列的一份子。因此只需要看`dp[i-1][j-1]`的值就行，即`text1[0:i-1]`与`text2[0:j-1]`的最长公共子序列的长度。由此可得：

`dp[i][j] = dp[i-1][j-1] + 1, text1[i] == text2[j]`

如示例一：

text1 = abcde
text2 = ace

当 i = 4，j = 2 时，就是上面所说的情况

如果`text[i]`不等于`text[j]`呢？

这时候两个元素没办法同时做贡献了，因此要么`text1[i]`做贡献，要么`text2[j]`做贡献

如果是`text1[i]`做贡献，那么`dp[i][j] = dp[i][j-1]`

如果是`text2[j]`做贡献，那么`dp[i][j] = dp[i-1][j]`

因为要求最长子序列，二者取最大值即可，即：

`dp[i][j] = max(dp[i-1][j], dp[i][j-1]), text1[i] != text2[j]`

可能的疑问：为什么不是等于`max(dp[i-1][j],dp[i][j-1],dp[i-1][j-1])`？

因为新遍历到的`text1[i]`或`text2[j]`，是有可能为“增长最长子序列”这件事情贡献力量的，只不过不能同时贡献力量罢了，举个例子：

text1 = aced

text2 = aefkc

此时 i = 2, j = 3，ace 与 aefk 的最长公共子序列为 a

现在 j 往后移动，尽管此时`text1[i] != text2[j]`，但因为 `text2[0:j]` 变为了 aefkc，新增加的 c 使得最长公共子序列变成了 ac

递推公式已经出来了，再确定临界情况

`dp[0][j]` 表示 `text1[0]` 和 `text2[0:j]` 构成的最长公共子序列的长度，因此只需要判断`text2[0:j]`中是否包含`text1[0]`，也就是判断遍历到的元素是否等于`text1[0]`即可

如果等于，就令当前j及其之后的所有`dp[0][j]`为1，终止循环；否则继续循环到底，然后令`dp[0][j]`为0

`dp[i][0]` 同理，遍历`text1[0:i]`，看是否有存在等于`text2[0]`的元素即可

Java

```java
class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length();
        int n = text2.length();
        int[][] dp = new int[m][n];

        //初始化 dp[i][0]
        for(int i = 0; i < m; i++) {
            if(text1.charAt(i) == text2.charAt(0)) {
                for(int k = i; k < m; k++) {
                    dp[k][0] = 1;
                }
                break;
            }
        }

        //初始化 dp[0][j]
        for(int j = 0; j < n; j++) {
            if(text2.charAt(j) == text1.charAt(0)) {
                for(int k = j; k < n; k++) {
                    dp[0][k] = 1;
                }
                break;
            }
        }

        for(int i = 1; i < m; i++) {
            for(int j = 1; j < n; j++) {
                if(text1.charAt(i) == text2.charAt(j)) {
                    dp[i][j] = dp[i-1][j-1] + 1;
                } else {
                    dp[i][j] = Math.max(dp[i][j-1], dp[i-1][j]);
                }
            }
        }
        return dp[m-1][n-1];
    }
}
```

当然，如果我们令`text1[0:i]`表示从下标0开始，长度为 i 的前缀，那么就不需要两个for循环这么繁琐的初始化工作了。因为空字符串与任何字符串的公共子序列长度都为0，所以无论是`dp[i][0]`还是`dp[0][j]`，都只需要初始化为0即可。只不过需要注意，开辟数组的时候，应该是(m+1)×(n+1)的二维数组，且关注的是`text1[i-1]`是否等于`text2[j-1]`

Python

```python
class Solution:
    def longestCommonSubsequence(self, text1: str, text2: str) -> int:
        m = len(text1)
        n = len(text2)

        dp = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(1, m + 1) :
            for j in range(1, n + 1) :
                if text1[i - 1] == text2[j - 1] :
                    dp[i][j] = dp[i - 1][j - 1] + 1
                else :
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
        
        return dp[m][n]
```
